Introduction

Some reasons why we choose/migrated to git

  • Advantages to individual developer workflows.
  • Advantages to team workflows.
  • Flexibility in workflow. Branches are very seamless.
  • Faster and less error prone than subversion is, no more are files locked, unable to be cleaned or updated.
  • The OSS tools we use all use git and workflows for submitting patches have moved to pull requests.

Goals

  • Mental Model

    Develop a mental model of the git repository, local vs remote, working directory vs stage vs index.

  • Visualize Commands

    Once we have a mental model we can examine how common git commands affect the model.

Graph Theory! Yay!

Mental Model

Graph

A set of object where some of the objects are connected by links. Real world you can think of Social Networks as graphs. Each person is a node and the relationship between them is an edge.

V={A,B,C,D,E,F,G,H}
E={{A,B},{A,C},{B,E},{B,F},{C,H},{D,G},{E,H},{F,H},{G,H}}

Arrows make Graphs Happy!

Mental Model

Directed Graph

Each edge is given a direction, that is the relationship between two objects/nodes/elements goes only one direction.

V={A,B,C,D,E,F,G,H}
E={{A,B},{A,C},{B,E},{B,F},{C,H},{D,G},{E,H},{F,H},{G,H}}

Acyclic Directed Graph

Takes a directed graph one step farther and says that there can be no way in the graph to ever start at one point and loop back to it.

E={{A,B},{A,C},{B,E},{B,F},{C,H},{D,G},{E,H},{F,H},{G,H}}

Commits, Ancestors, Changes, Oh My!

Mental Model

git Commit

A commit represents not only the changes that have been made, who did it, when and the commit message. But a commit also represents the relationship to past commits as well. This is really important to understand as we start thinking about merges, rebases, and history.
